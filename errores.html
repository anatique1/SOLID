Errores y malas implementaciones del principio ISP

Error 1: Interfaces muy grandes

Cuando una interfaz tiene muchos metodos, algunas clases se ven obligadas a usar cosas que no necesitan.

Codigo incorrecto:

class Trabajador:
    def trabajar(self):
        pass
    
    def comer(self):
        pass

class Robot(Trabajador):
    def trabajar(self):
        print("Robot trabajando")

    def comer(self):
        pass

Forma correcta:

class Trabajador:
    def trabajar(self):
        pass

class Robot(Trabajador):
    def trabajar(self):
        print("Robot trabajando")

Error 2: Metodos que no se usan

Algunas clases tienen metodos que no usan, lo que hace el codigo mas confuso.

Codigo incorrecto:

class Dispositivo:
    def imprimir(self):
        pass

    def escanear(self):
        pass

class Impresora:
    def imprimir(self):
        print("Imprimiendo")

    def escanear(self):
        pass

Forma correcta:

class Impresora:
    def imprimir(self):
        print("Imprimiendo")

Error 3: Dependencias innecesarias

Una clase usa mas cosas de las que realmente necesita.

Codigo incorrecto:

class Multifuncional:
    def imprimir(self):
        pass
    
    def escanear(self):
        pass

class Usuario:
    def usar(self, maquina):
        maquina.imprimir()

Forma correcta:

class Impresora:
    def imprimir(self):
        pass

class Usuario:
    def usar(self, impresora):
        impresora.imprimir()

Conclusion

El error mas comun es crear interfaces muy grandes y obligar a las clases a implementar metodos que no necesitan. Lo mejor es dividir las interfaces.
